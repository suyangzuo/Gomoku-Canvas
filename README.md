# Gomoku Canvas

一个基于 HTML5 Canvas 构建的五子棋（Gomoku）游戏，支持高清渲染、实时悬停预览与响应式布局。

## 项目结构

```
Gomoku-Canvas/
├── index.html          # 页面入口
├── index.css           # 样式表
├── index.js            # 游戏核心逻辑（Canvas 渲染 + 交互）
├── Images/
│   ├── 黑白棋子.webp   # 棋子精灵图（黑子 / 白子）
│   └── Logo.webp       # 站点图标
└── README.md           # 本文件
```

## 技术栈

- **HTML5 Canvas 2D API** — 棋盘、棋子、索引及悬停效果全部通过 Canvas 绘制
- **ResizeObserver** — 监听视口变化，自动重绘棋盘与已落棋子
- **CSS Flexbox** — 画布与侧边面板的响应式布局
- **Google Fonts** — Google Sans Code、Noto Sans SC、JetBrains Mono 等字体

## 核心特性

| 特性 | 说明 |
|------|------|
| 高清渲染 | 通过 `devicePixelRatio` 与 `ctx.scale()` 实现物理像素级清晰绘制 |
| 精灵图绘制 | 使用单张 WebP 精灵图绘制黑白棋子，减少 HTTP 请求 |
| 响应式棋盘 | 窗口缩放时自动重算棋盘尺寸、重绘所有已落棋子 |
| 悬停预览 | 鼠标悬停时显示十字高亮与行列索引高亮；移出后精确擦除 |
| 步数序号 | 每步落子在棋子上方标注当前步数，便于复盘 |
| 落子记录 | 内部维护 `对弈记录` 数组，支持后续扩展悔棋、复盘等功能 |

## 快速开始

本项目为纯前端静态页面，无需构建步骤，直接在浏览器打开即可。

### 本地运行

```bash
# 方式一：直接打开
open index.html

# 方式二：使用任意本地服务器（推荐，避免图片跨域问题）
npx serve .
# 或
python -m http.server 8000
```

然后在浏览器访问 `http://localhost:8000`。

## 游戏操作

1. 点击「开始」按钮进入对局状态
2. 鼠标悬停棋盘格查看落子预览
3. 点击棋盘格落子，黑白双方交替
4. 点击「停止」可随时结束当前对局并重置

## 代码架构

### `class 五子棋`

| 方法 | 职责 |
|------|------|
| `constructor()` | 初始化画布、加载棋子精灵图、绑定事件（图片加载完成后才真正初始化棋盘） |
| `刷新棋盘数据()` | 根据当前画布尺寸计算棋盘格宽度、棋子半径，并重绘棋盘 |
| `绘制棋盘()` | 绘制经纬线与行列索引 |
| `绑定事件()` | 处理 `mousemove`、`mousedown`、`scroll` 事件 |
| `绘制悬停预览效果()` | 悬停时绘制十字高亮与索引高亮 |
| `移除预览效果()` | 精确擦除悬停预览，恢复底层经纬线 |
| `绘制棋子()` | 从精灵图截取对应颜色棋子并绘制到指定格 |
| `绘制棋子序号()` | 在已落子位置标注当前步数 |
| `保存对弈记录()` | 将落子信息写入内部数组 |
